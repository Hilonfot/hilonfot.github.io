程序员七问

发表于 2018-04-12 | 分类于 闲谈 |

有不少人向我抱怨说工作中都是写业务代码，刚开始还有新鲜感，熟练了以后就觉得无聊了。

这样的问题多了，也促使我去思考和总结，我总结了一个自检的列表， 如果你已经确定在一个公司发展下去，不妨对照着检查一下，看看差距在哪里，估计就不会无聊了。

1. 我是否对系统的业务有了整体的了解？

我能不能对其他人（例如面试官）描述下该系统实现的业务：

系统有哪些角色，这些角色如何与系统交互；

系统有哪些主要流程，次要流程，都有哪些角色参与其中。

2. 系统的整体架构是什么样子？

系统都分为哪些组件，这些组件是如何部署在服务器/客户端的？

它们之间是怎么交互的？用的什么协议？

3. 系统用到了哪些技术和框架，我是不是都已经精通了？

例如Spring, Redis, Ngnix……这些框架和软件的原理都搞清楚了吗？

4. 我是不是已经精通了多个模块甚至整个系统的代码？

对于一个bug， 我能否迅速地定位到bug的源头？

对于“烂代码”，我是否有能力，有勇气去重构它？

服务器安全注意事项

发表于 2018-01-23 | 分类于 服务器 |

PHP版本5.5（性能）

网站运行目录权限（防止www运行被上传目录）常用目录
httpd禁止列出目录（防止攻击者直接查看到目录树对应东西）
关闭php高危系统函数（防止系统隐患）

绝对禁止

*.gif等

禁止在版本库里面写任何账户相关东西
当被攻击后，攻击者可以直接在版本库中发现相关用户密码 mysql alipay wxpay 等密码
当发现被攻击后，需要全部重新生成 shh公钥，防止攻击者利用ssh下载代码,修改密码
数据库连接禁止使用root用户，root是mysql中的超级用户，
数据库中尽量使用 一户一库，尽量避免用户 Host’%’带外连接
开启防火墙，禁止非必要连接，进/出服务器（禁止外部ping可选）
redis必须授权访问
尽量减少开发过程的面向过程开发，加入框架统一安全规则过滤，开发时候尽量使用框架的图片上 传，GET POST REQUEST COOKIE过滤
在开发过程中，尽量使用单一入口文件,单一配置文件。配置文件权限尽量使用rwx000000(700权限) 防止帐号泄漏
在开发过程中，尽量避免在项目加入 /phpinfo.php /test.php /testmysql.php /plugs/ 等攻 击者能猜出用途的东西，增加被猜出的难度
技术开发人员在日常工作和生活过程中，禁止向非公司人员透露人和技术细节。避免别有用心的攻 击者盯上，或者得道攻击便利
服务器日常备份，尽量有专人、专机负责避免数据丢失
尽量减少系统中ftp使用，推荐使用sftp ssh等实现文件上下载
尽量让有限的用户拥有服务器权限，对于临时服务器权限，要及时回收。
服务器中需要必要的安全软件(linux安骑士等 window360卫士等),开启必要的软件防护
当发现系统中有被攻击者攻击时候，要即使修复或者替换被串改的程序（此时杜绝使用被修改过的 程序，尤其是动态库.dll,.exe,linux下的执行文件），避免被木马盗取帐号信息
尽量避免在服务器中使用浏览器，防止被挂马
修改常用端口(20,21,22,25等)

Git的小技巧🙈

Git常用命令集合，Fork于tips项目

Git是一个“分布式版本管理工具”，简单的理解版本管理工具：大家在写东西的时候都用过“回撤”这个功能，但是回撤只能回撤几步，假如想要找回我三天之前的修改，光用“回撤”是找不回来的。而“版本管理工具”能记录每次的修改，只要提交到版本仓库，你就可以找到之前任何时刻的状态（文本状态）。

下面的内容就是列举了常用的Git命令和一些小技巧，可以通过”页面内查找”的方式进行快速查询：Ctrl/Command+f。

开卷必读

如果之前未使用过Git，可以学习Git小白教程入门

  1. 一定要先测试命令的效果后，再用于工作环境中，以防造成不能弥补的后果！到时候别拿着砍刀来找我
  2. 所有的命令都在git version 2.7.4 (Apple Git-66)下测试通过
  3. 统一概念：
    • 工作区：改动（增删文件和内容）
    • 暂存区：输入命令：git add 改动的文件名，此次改动就放到了‘暂存区’
    • 本地仓库(简称：本地)：输入命令：git commit 此次修改的描述，此次改动就放到了’本地仓库’，每个commit，我叫它为一个‘版本’。
    • 远程仓库(简称：远程)：输入命令：git push 远程仓库，此次改动就放到了‘远程仓库’（GitHub等)
    • commit-id：输出命令：git log，最上面那行commit xxxxxx，后面的字符串就是commit-id
  4. 如果喜欢这个项目，欢迎Star、提交Pr、反馈问题😊
